# Heuristic Algorithms 教学实验项目 PRD

## 1. 项目概述

本项目旨在在当前仓库中构建一套系统、清晰、可运行、可视化效果强的启发算法教学实验内容。最终产出不只是一些能跑的算法代码，而是一套适合学习、展示、授课和复现实验的 Notebook 集合。

这套内容将延续你之前 RL 项目的组织思路，但主题切换为“启发算法”。整体目标是让读者能够从直觉理解、经典案例、算法实现、实验过程和可视化结果几个层面，真正看懂每一种算法。

## 2. 项目目标

本项目需要完成以下目标：

1. 提供一份总览型教学文档，解释为什么做这套内容、覆盖哪些算法、如何组织实验。
2. 为每一种启发算法建立独立目录和独立 `ipynb`。
3. 每个 Notebook 都要使用经典案例，而不是只展示抽象公式。
4. 每个 Notebook 都要有高质量可视化，能够展示算法“如何搜索、如何变化、如何收敛”。
5. 所有实验内容要结构统一，便于横向比较不同算法。

## 3. 项目定位

本项目不是论文式推导，也不是工程性能竞赛，而是一套“教学导向”的算法实验手册。

它强调以下几点：

- 先让读者看懂，再谈优化细节。
- 用经典问题承载算法，而不是只写抽象函数。
- 用可视化解释搜索过程，而不是只给最终答案。
- 保证 Notebook 自包含，可单独打开和运行。

## 4. 目标读者

本项目主要面向以下读者：

1. 初学启发算法的学生
2. 想快速建立算法直觉的开发者
3. 需要拿 Notebook 做展示、授课或分享的人
4. 希望后续继续扩展算法实验的人

因此，文档和 Notebook 的写法都要尽量清楚、直接、分层明确，避免只对“已经懂的人”友好。

## 7. 算法范围

本次项目计划覆盖 12 个核心启发算法或元启发算法。这一组内容已经足够构成一套结构完整、适合教学的启发算法实验集。

### 7.1 本阶段必做算法

1. Hill Climbing
2. Random Restart Hill Climbing
3. Simulated Annealing
4. Tabu Search
5. Genetic Algorithm
6. Evolution Strategy
7. Differential Evolution
8. Particle Swarm Optimization
9. Ant Colony Optimization
10. Artificial Bee Colony
11. GRASP
12. Iterated Local Search

### 7.2 为什么选择这 12 个

选择标准如下：

1. 经典性强
   - 这些算法是启发算法教学中最常见的一批。
2. 类型覆盖广
   - 包含局部搜索、随机搜索、群体智能、进化算法等不同思路。
3. 适合做可视化
   - 这些算法大多能通过轨迹、路径、收敛过程、种群演化等形式直观展示。
4. 适合做对比教学
   - 不同算法可以共享一部分问题场景，便于横向比较。

## 8. 教学设计原则

整套内容需要遵循以下教学原则。

### 8.1 先建立直觉，再展示代码

每个 Notebook 都应该先说明：

- 这个算法试图解决什么问题
- 它和其他算法相比，核心思路有什么不同
- 它为什么可能有效

不能一上来就塞代码，否则教学效果会很差。

### 8.2 先给经典案例，再谈一般化

对初学者来说，“算法运行在一个具体问题上”比“算法适用于很多问题”更容易理解。因此每个 Notebook 都应先绑定一个经典案例，用实验把算法讲清楚，再补充它还能用在哪些地方。

### 8.3 可视化要服务于解释

可视化不是为了堆图，而是为了回答问题：

- 算法如何移动？
- 算法如何避免陷入局部最优？
- 算法如何从很多候选解中逐步收敛？
- 算法的参数变化会带来什么不同结果？

### 8.4 每个 Notebook 要自成体系

读者应该可以单独打开任意一个 `ipynb`，不依赖其他 Notebook，也能完整理解该算法的基本逻辑和实验结果。

## 9. 问题类型与案例选择策略

为了让不同算法的优势和行为清楚可见，本项目不会强行把所有算法套到同一个问题上，而是按照算法特性选择最适合的经典案例。

### 9.1 连续优化问题

适合用二维函数地形做展示，例如：

1. Sphere
2. Rosenbrock
3. Rastrigin
4. Ackley
5. Himmelblau

这类问题的优点是：

- 可以画等高线图
- 可以画搜索轨迹
- 可以展示局部最优与全局最优
- 适合粒子、种群、轨迹类算法

### 9.2 组合优化问题

适合用离散结构问题做展示，例如：

1. Traveling Salesman Problem, TSP
2. N-Queens
3. Knapsack
4. Set Cover
5. 简化调度问题

这类问题的优点是：

- 案例经典
- 结果形态清晰
- 容易体现路径搜索、邻域变换、构造策略等过程

## 10. 算法与经典案例映射

为了保证展示效果，建议采用如下映射。

### 10.1 01 Hill Climbing

- 问题：Himmelblau 或 Rastrigin
- 目标：展示贪心上山策略的优点与局部最优缺点
- 适合可视化：二维等高线图、搜索轨迹图、不同初始点结果对比图

### 10.2 02 Random Restart Hill Climbing

- 问题：Rastrigin
- 目标：展示重启如何缓解局部最优问题
- 适合可视化：多次重启轨迹图、最终结果分布图、最优值比较图

### 10.3 03 Simulated Annealing

- 问题：TSP
- 目标：展示“允许差解被接受”如何帮助跳出局部最优
- 适合可视化：路径演化图、温度下降曲线、接受率变化图、收敛曲线

### 10.4 04 Tabu Search

- 问题：TSP 或 N-Queens
- 目标：展示禁忌表如何防止短期来回震荡
- 适合可视化：解的变化图、最优值曲线、禁忌记忆影响示意图

### 10.5 05 Genetic Algorithm

- 问题：TSP
- 目标：展示种群演化、选择、交叉、变异如何共同改进解
- 适合可视化：种群适应度分布图、最优个体变化图、路径演化图

### 10.6 06 Evolution Strategy

- 问题：Rosenbrock
- 目标：展示进化策略在连续空间中的搜索方式
- 适合可视化：父代/子代分布图、步长变化图、收敛轨迹图

### 10.7 07 Differential Evolution

- 问题：Ackley 或 Rastrigin
- 目标：展示差分变异带来的探索能力
- 适合可视化：种群位置图、迭代分布变化图、收敛曲线

### 10.8 08 Particle Swarm Optimization

- 问题：Himmelblau
- 目标：展示粒子如何受个体最优和群体最优共同驱动
- 适合可视化：粒子轨迹图、速度变化图、全局最优变化图

### 10.9 09 Ant Colony Optimization

- 问题：TSP
- 目标：展示信息素累积如何引导更优路径出现
- 适合可视化：路径构造图、信息素热力图、收敛曲线

### 10.10 10 Artificial Bee Colony

- 问题：Sphere 或 Ackley
- 目标：展示雇佣蜂、观察蜂、侦察蜂的分工搜索机制
- 适合可视化：种群分布图、阶段变化图、收敛曲线

### 10.11 11 GRASP

- 问题：Set Cover 或简化 TSP
- 目标：展示“贪心随机化构造 + 局部搜索”两阶段策略
- 适合可视化：构造过程图、改进前后对比图、多次运行结果图

### 10.12 12 Iterated Local Search

- 问题：TSP
- 目标：展示局部搜索、扰动和再优化的循环机制
- 适合可视化：扰动前后路径图、局部最优跳出过程图、收敛曲线

## 11. 每个 Notebook 的统一教学结构

为保证一致性，所有 Notebook 使用统一章节设计。这样读者在切换算法时，不需要重新适应阅读方式。

建议结构如下。

### 11.1 标题与摘要

内容包括：

- 算法名称
- 中文名称
- 一句话概括算法思想
- 本 Notebook 使用的经典案例

### 11.2 为什么要学这个算法

这一节回答：

- 这个算法解决什么问题
- 它和前面算法有什么不同
- 它在启发算法体系中的位置是什么

### 11.3 问题定义

明确本 Notebook 里的实验问题：

- 输入是什么
- 输出是什么
- 优化目标是什么
- 如何评价解的好坏

### 11.4 算法直觉

用自然语言说明：

- 算法如何搜索
- 为什么这样设计
- 关键机制是什么

这一节是教学重点，尽量少堆公式，优先讲清直觉。

### 11.5 算法步骤

用结构化方式列出：

1. 初始化
2. 候选解生成
3. 评价
4. 更新
5. 终止条件

必要时可补简化伪代码。

### 11.6 从零实现

实现要求：

- 代码可读
- 变量命名清晰
- 关键步骤有必要注释
- 避免为了“工程封装”牺牲教学可读性

### 11.7 单次实验演示

这一节的作用是让读者亲眼看到：

- 算法从什么状态开始
- 过程中如何变化
- 最后得到什么结果

### 11.8 可视化分析

这一节必须是重点，不是附属。

要展示：

- 搜索过程
- 解的结构变化
- 收敛趋势
- 算法特有状态变化

### 11.9 参数分析

选 1 到 3 个关键参数做敏感性实验，例如：

- SA：初始温度、降温率
- GA：种群大小、变异率
- PSO：惯性权重、学习因子
- ACO：蒸发率、信息素权重

### 11.10 小结

总结：

- 这个算法最适合什么情况
- 它的优点和局限
- 和其他算法相比最突出的特点

## 12. 可视化标准

本项目强调“非常好的可视化”，因此每个 Notebook 不能只画默认折线图，而应按照统一标准设计展示内容。

### 12.1 所有 Notebook 必须至少包含三类图

1. 收敛图
   - 展示最优值或目标值如何随迭代变化
2. 过程图
   - 展示搜索过程中的关键状态变化
3. 结果图
   - 展示最终解的形态或最终搜索位置

### 12.2 推荐的算法特定可视化

#### 局部搜索类

- 地形等高线图
- 单步轨迹图
- 多次初始点对比图

#### 进化算法类

- 种群散点图
- 适应度分布图
- 各代最优值变化图

#### 群体智能类

- 粒子或个体运动轨迹图
- 信息素热力图
- 群体分布演化图

#### 组合优化类

- 路径图
- 邻域变化前后对比图
- 多轮迭代路径变化图

### 12.3 图表质量要求

图表应满足以下要求：

1. 标题明确，不使用模糊命名
2. 坐标轴标签完整
3. 图例清楚
4. 颜色有语义
5. 关键结果应有标注
6. 布局尽量整洁，不显得拥挤

## 13. 实验设计要求

每个 Notebook 不能只运行一次并展示一个结果，还要体现算法行为的稳定性和参数影响。

### 13.1 单次实验

要求：

- 固定随机种子
- 展示中间状态
- 保证读者能跟踪算法过程

### 13.2 多次实验统计

要求：

- 多次重复运行
- 统计均值、最优值、最差值、标准差
- 用图表展示波动和稳定性

### 13.3 参数敏感性分析

要求：

- 至少选一组核心参数
- 展示参数变化后性能差异
- 给出简短解释

## 14. 风格与写作要求

因为这是教学内容，Notebook 的文字风格也需要统一。

### 14.1 写作原则

1. 解释优先
2. 术语准确
3. 句子简洁
4. 避免无解释的大段公式
5. 结论要和图表相呼应

### 14.2 代码原则

1. 优先清晰，不追求炫技
2. 关键步骤拆成独立函数
3. 少量必要注释
4. 随机过程固定种子以便复现
5. 避免让 Notebook 依赖复杂外部环境

## 18. 风险与应对

### 18.1 风险

1. 算法数量较多，后期容易出现质量不一致
2. 某些算法如果案例选得不好，可视化效果会不足
3. 如果每个 Notebook 都写得过于复杂，教学反而会变差

### 18.2 应对策略

1. 统一模板先行
2. 先做高质量样板，再扩展
3. 优先保证可读性，而不是过度堆功能
4. 让每个 Notebook 围绕一个最经典、最适合展示的案例展开

## 19. 后续执行顺序

如果后续执行得当，这套内容最终会形成：

1. 一份清晰的总设计文档

这将使整个项目更像一门“可操作的启发算法课程”，而不是零散代码集合。
